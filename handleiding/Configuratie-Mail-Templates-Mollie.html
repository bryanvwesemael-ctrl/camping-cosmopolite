Deel 1

E-mailconfiguratie (Resend)

Resend is de dienst die de mails effectief verstuurt. Gratis tot 3.000 mails per maand — ruim voldoende voor een camping.

Stap A — Account aanmaken

  1. Ga naar resend.com en maak een gratis account aan
  2. Bevestig je e-mailadres

Stap B — Je domein verifiëren (belangrijkste stap!)

Om mails te sturen vanaf info@campingcosmopolite.be moet je bewijzen dat je dat domein bezit. Anders belanden de mails in de spam of worden ze geweigerd.

  1. In Resend: ga naar Domains → Add Domain
  2. Vul je domein in (bv. campingcosmopolite.be)
  3. Resend toont enkele DNS-records (TXT/CNAME)
  4. Voeg die records toe bij je domeinbeheerder (waar je het domein kocht)
  5. Wacht tot Resend "Verified" toont (kan tot enkele uren duren)
⚠️ Geen eigen domein?Heb je (nog) geen eigen domein, dan kan je tijdelijk testen met het standaard testadres van Resend. Maar voor échte gasten heb je een geverifieerd domein nodig — anders komen mails niet of in de spam aan.

Stap C — API-sleutel aanmaken

  1. In Resend: ga naar API Keys → Create API Key
  2. Geef een naam (bv. "Camping dashboard") en kies Sending access
  3. Kopieer de sleutel — ze begint met re_
⚠️ Eénmalig zichtbaarResend toont de sleutel maar één keer. Kopieer ze meteen. Ben je ze kwijt, maak dan gewoon een nieuwe aan.

Stap D — Invullen in het dashboard

⚙️ Instellingen → Mail
  1. Vul bij Afzender → Naam in: Camping Cosmopolite
  2. Vul bij Afzender → E-mail je geverifieerde adres in (bv. info@campingcosmopolite.be)
  3. Plak je Resend-sleutel bij Resend API Key
  4. Klik op opslaan
💡 Afzender-e-mail = geverifieerd domeinHet e-mailadres bij "Afzender" moet op hetzelfde domein zitten dat je in stap B verifieerde. Anders weigert Resend te versturen.

Deel 2

Mailtemplates

Templates zijn de standaardteksten die je hergebruikt. Je stelt ze één keer in en gebruikt ze telkens opnieuw.

📧 Mail (bovenaan) → tabblad Templates

De types

TypeWanneer?
BevestigingWanneer je een aanvraag bevestigt
HerinneringEen paar dagen voor aankomst
BetalingOm aan een openstaande betaling te herinneren
UitcheckenBij of na vertrek (bv. bedankt + review-vraag)

Variabelen — automatisch invullen

Gebruik variabelen in je tekst; ze worden automatisch vervangen door de echte gegevens van de gast.

VariabeleWordtVariabeleWordt
{{voornaam}}Voornaam{{personen}}Aantal personen
{{aankomst}}Aankomstdatum{{bedrag}}Totaalbedrag
{{vertrek}}Vertrekdatum{{ogm}}Betaalreferentie
{{nachten}}Aantal nachten
💡 Voorbeeld"Beste {{voornaam}}, je verblijf van {{aankomst}} tot {{vertrek}} ({{nachten}} nachten) is bevestigd. Totaal: {{bedrag}}." → wordt automatisch: "Beste Jan, je verblijf van 26 jun tot 28 jun (2 nachten) is bevestigd. Totaal: €178,50."

Meerdere varianten = natuurlijker

Voeg per type meerdere varianten toe. Bij het versturen kiest het systeem er willekeurig één. Zo klinken je mails nooit twee keer identiek — persoonlijker, en minder kans op spam.

Een template bewerken

  1. Ga naar 📧 Mail → Templates
  2. Klik op een variant om ze open te klappen
  3. Pas onderwerp en inhoud aan (gebruik gerust variabelen)
  4. Klik op 👁 Preview om de mail met voorbeeldgegevens te zien
  5. Klik op + Variant om een extra versie toe te voegen
  6. Klik op Opslaan
💡 Goede mailsHou het kort en vriendelijk. Begin met {{voornaam}}, zet de belangrijkste info (datum, bedrag, referentie) duidelijk, en eindig met je campingnaam. Gebruik de Preview vóór je opslaat.

Deel 3

Mollie betalingen

Mollie regelt online betalingen (Bancontact, kaart, ...). Gasten krijgen een betaallink; zodra ze betalen, springt de boeking automatisch op "betaald".

Stap A — Account aanmaken

  1. Ga naar mollie.com en maak een zakelijk account aan
  2. Vul je bedrijfsgegevens in (KBO, rekeningnummer...) voor de verificatie
⚠️ Verificatie duurt evenMollie controleert je bedrijfsgegevens voor je échte betalingen kan ontvangen. Start dit tijdig op. Ondertussen kan je al testen met de test-sleutel (zie hieronder).

Stap B — De API-sleutel ophalen

Mollie heeft twee soorten sleutels:

SleutelBegint metGebruik
Testtest_Om uit te proberen — geen echt geld
Livelive_Echte betalingen (na verificatie)
  1. In Mollie: ga naar Developers → API keys
  2. Kopieer eerst je test-sleutel (test_...)
  3. Later, na verificatie, vervang je die door de live-sleutel

Stap C — Invullen in het dashboard

⚙️ Instellingen → Mollie
  1. Plak je Mollie-sleutel in het veld
  2. Klik op Sleutel opslaan
💡 Begin altijd met de test-sleutelTest eerst de volledige flow met test_. Pas als alles klopt, schakel je over naar de live_-sleutel.

Stap D — De webhook (technisch, eenmalig)

Opdat een betaling automatisch als "betaald" wordt geregistreerd, meldt Mollie dit terug aan het systeem via een webhook. Die moet ingesteld zijn op de server.

⚠️ Belangrijk veiligheidspuntDe webhook mag uitsluitend via de beveiligde serversleutel (service_role) werken — nooit via de publieke sleutel. Zo kan niemand zelf een boeking op "betaald" zetten zonder te betalen. Laat dit door je ontwikkelaar nakijken vóór je live gaat.

Een betaling versturen (in de praktijk)

  1. Open een boeking → klik op 💳 Betalen
  2. Bevestig — er wordt een betaallink aangemaakt
  3. Stuur die link naar de gast (mail of bericht)
  4. Na betaling springt de status automatisch op "betaald"
